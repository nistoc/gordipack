### Фаза 1 — Двойная запись (COORD)
COORD при каждом тике:
1. Читает из md (как раньше)
2. **Дополнительно** пишет в `mezosync.db` (INSERT INTO messages)
3. Md остаётся источником правды

Длительность: 2-3 дня, пока все роли не перейдут.

### Фаза 2 — Двойная запись (все роли)
Каждая роль при ребёрсе получает в launcher-промпте инструкцию:
- Читать из SQLite (primary) + md (fallback)
- Писать в оба

### Фаза 3 — SQLite = primary
- Чтение только из SQLite
- Md генерируется export-markdown.py (read-only для людей)
- Phoenix хранится в таблице phoenix (не в .md)

### Фаза 4 — Md отключены
- sync.*.md больше не пишутся
- phoenix.*.md генерируется из БД только по запросу владельца
- Полный переход завершён

## Что НЕ меняется
- Git-коммиты координационных файлов — COORD продолжает коммитить
- Правило push — без слова владельца
- Таймерный механизм (ScheduleWakeup) — остаётся в ведении хоста
- Viewer (index.html) — открывается локально, drag-and-drop .db файла
