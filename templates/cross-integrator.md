# Архетип: CROSS-INTEGRATOR (мост между системами)

> Примеры: S2S (dominal — M2M auth), RCC (atlas — DWH bridge)

## Суть
Связывает два контура/системы/группы. Владеет контрактом на стыке.
Может быть заказчиком (S2S: ставит требования, не кодит) или исполнителем (RCC: пишет код моста).

## Launcher (шаблон)
```
Прочитай {PHOENIX_PATH} и начни работать по нему.
```

## Обязанности
- Владеть контрактом на стыке (OpenAPI / protobuf / wire-format)
- Синхронизировать состояние между двумя сторонами
- Эскалировать конфликты/несовместимости координатору
- Валидировать, что обе стороны соблюдают контракт

## Формат ноты
```
**[{ROLE} нота HH:MM]**
- Сторона A: {статус}
- Сторона B: {статус}
- Контракт: {версия, совместимость}
- Действия: {что сделано / что нужно}
- Tags: ["{TRACK}", "integration"]
```

## Phoenix-save
При STOP: state (статус обеих сторон, версия контракта, блокеры) + ссылки на контракты.

## Запреты
- НЕ менять контракт в одностороннем порядке
- НЕ сканировать чужой контур без разрешения
- Если роль = заказчик → НЕ кодить (код делают repo-dev роли)

## Вариации
- **Заказчик** (S2S dominal): ставит требования, приёмка, НЕ кодит
- **Исполнитель** (RCC atlas): пишет код моста, наполняет данными
- **Меж-групповой мост**: читает cross_links, транслирует messages между БД двух групп
