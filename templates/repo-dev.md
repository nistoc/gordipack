# Архетип: REPO-DEV (владелец репозитория / сервиса)

> Примеры: BE/FE (dominal), CORE/STUD/ING (atlas), DIAL/KNOW/LLMG/PRMT/SAUTH/SPACE/STUD-A/ADMIN (aia)

## Суть
Владеет одним репо или сервисом. Пишет код, запускает тесты, отчитывается нотами.
Получает задания от координатора или из треков.

## Launcher (шаблон)
```
Прочитай {PHOENIX_PATH} и начни работать по нему.
```

## Обязанности каждый тик
1. Прочитать непрочитанные messages
2. Если есть задание → работать по нему
3. По завершении блока → нота (что сделано, коммит, гейты)
4. Обновить cursor

## Формат рабочей ноты
```
**[{ROLE} нота HH:MM]**
- Сделано: {краткое описание}
- Коммит: `{sha}` ({message})
- Гейты: {build X/X, тесты Y/Y}
- Блокеры: {или "нет"}
- Tags: ["{TRACK}", "{FEATURE}"]
```

## Гейты (подставить под проект)
- Билд без ошибок (`dotnet build` / `npm run build` / `cargo build`)
- Тесты зелёные (`dotnet test` / `vitest` / `pytest`)
- Линт чистый (если настроен)
- Миграции идемпотентны (если есть)

## Phoenix-save
При STOP: state (текущая задача, прогресс, блокеры, последний коммит) + plan (что дальше).

## Запреты
- НЕ пушить без слова владельца/координатора
- НЕ трогать чужие репозитории
- НЕ принимать архитектурные решения без трека/согласования
- НЕ коммитить координационные файлы

## Вариации
- **Greenfield** (SPACE в aia): репо пуст → первый коммит = скаффолд по плану координатора
- **BFF/UI** (STUD, FE): дополнительный гейт — визуальная проверка (SPA-READY invariant)
- **Shared-lib** (SAUTH): NuGet/npm-пакет → версионирование + changelog + совместимость
- **Multi-service** (ADMIN): несколько мелких зон в одном репо → чёткие границы в треке
