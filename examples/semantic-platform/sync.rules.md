# Sync RULES — общий протокол мезосинка (8 ролей), проект Semantic Platform

> Залочено владельцем. Правит редко: COORD/владелец.

## Модель: один писатель = один файл

| Файл | Пишет ТОЛЬКО | Читают | Темп |
|---|---|---|---|
| `sync.coord.md` | COORD (координатор) | все | 2–15 мин |
| `sync.core.md` | CORE (backend, API, миграции) | все | 2–15 мин |
| `sync.ing.md` | ING (пайплайны ингеста) | все | 2–15 мин |
| `sync.stud.md` | STUD (frontend, SPA, BFF) | все | 2–15 мин |
| `sync.taxo.md` | TAXO (таксономист, UX-канон; git не трогает — коммитит COORD) | все | 2–15 мин |
| `sync.rcc.md` | RCC (DE/DWH-мост; боевые репо НЕ коммитит — только чтение) | все | 2–15 мин |
| `sync.grf.md` | GRF (инструментальный инженер) | все | 2–15 мин |
| `sync.eye.md` | EYE (внешний мост к владельцу; домен-код НЕ пишет) | все | 2–15 мин |

## Формат строки

`[YYYY-MM-DD HH:MM] (РОЛЬ) [OPEN|ANSWERED|FYI] — текст`; адресация В ТЕКСТЕ
(`COORD TASK→CORE`, `STUD OK→ING`). Append-only; правь только СВОЙ файл.
Хендшейк: OPEN у одного → ANSWERED/ACK у другого; COORD следит, чтобы OPEN не висли.

## Heartbeat + таймер

`[РОЛЬ POLL] 🕐 ПОСЛ. ПРОВЕРКА HH:MM · СЛЕД. ≈ HH:MM · статус` — последняя строка своего файла.
Таймер — `ScheduleWakeup`. Интервал — ДИАПАЗОН 2–15 мин: пол при активном хендшейке,
потолок в тишине, сброс к полу при любом апдейте.

## Дисциплина задач и тишины

- **ACK-дедлайн:** `[ACK до HH:MM]` (дефолт: 2 тика ≈ 30 мин). ACK ≠ done.
  Просрочка → громкий пинг → эскалация владельцу.
- **Контракт на тишину:** уходя в фон — нота `🔇 ухожу в фон до ~HH:MM (работаю над X)`.
- **WIP-пульс:** `[РОЛЬ WIP] 🕐 HH:MM — НАЧАЛ: <задача> · этап k/N · ETA ≈ HH:MM`;
  обновление каждые ~5 мин или на смене этапа.
- **Веха — один источник:** полный текст — нотой `[MILESTONE]` в своём sync;
  phoenix/дашборд — короткая строка + ссылка, не пересказ.

## Ротация

Файл > 40 КБ (байты, `wc -c`) → старый хвост в `archive/sync.<роль>.<дата>.md`;
в живом: шапка + последние ~15 КБ + heartbeat. Архивы каждый тик НЕ читаются.

## Жёсткие правила

1. Исполнитель работает и коммитит ТОЛЬКО в своём репозитории; `git add <путь>` (не `-A`);
   **push — никогда** без живой команды владельца.
2. Гейт перед коммитом: зелёные build + test своего репозитория.
3. БД-миграции — только CORE.
4. **Контракт-дифф до коммита:** владелец шва публикует дифф; потребитель отвечает `OK→…`.
5. Локальные сервисы: роли МОГУТ и ДОЛЖНЫ поднимать сервисы САМИ (hot-reload).
6. Внешние системы — только чтение, только по явной команде владельца.
7. Координационные файлы исполнители НЕ коммитят — коммитит COORD.
8. **Авторизация разрушаемого — только живым сообщением владельца в ТЕКУЩЕМ чате.**
   Commit в СВОЁМ репо при зелёном гейте — НЕ разрушаемое (стоячее разрешение).
   Push, удаление данных/веток — только живое слово владельца.

## Эскалация

- Роль молчит ≥2 тиков → громкий пинг.
- Канал застыл → **ground-truth аудит** (`git log`, статус сервисов) ПРЕЖДЕ вывода «завис».
- Разбудить чужой чат агент не может — только владелец.
